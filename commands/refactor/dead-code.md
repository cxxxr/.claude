---
description: デッドコード（未使用コード）を検出して削除
argument-hint: "[path]"
---

指定されたパス（デフォルト: プロジェクト全体）でデッドコードを検出し、削除してください。

## 検出対象

1. **未使用の変数・定数** - 宣言されているが参照されていない
2. **未使用の関数・メソッド** - 定義されているが呼び出されていない
3. **未使用のインポート** - importされているが使用されていない
4. **未使用の型定義** - 定義されているが参照されていない
5. **到達不能コード** - return/throw後のコード、常にfalseの条件分岐内のコード
6. **コメントアウトされたコード** - 古いコードがコメントとして残っている

## 手順

1. まず対象パスのコードを分析し、デッドコードの候補をリストアップ
2. 各候補について、本当に未使用かを確認（間接参照、リフレクション、動的呼び出しに注意）
3. 削除しても安全なものを特定し、ユーザーに確認を求める
4. 承認後、デッドコードを削除
5. 削除後、ビルド/リント/テストが通ることを確認

## 注意事項

- エクスポートされたAPIは外部から使用される可能性があるため慎重に判断
- テストコードは本番コードから参照されないが削除しない
- 設定ファイルで参照される可能性のあるコードに注意
- 削除前に必ず確認を取る
