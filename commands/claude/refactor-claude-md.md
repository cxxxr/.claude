# CLAUDE.md リファクタリング

現在のプロジェクトの CLAUDE.md ファイルを分析し、ベストプラクティスに基づいてリファクタリングしてください。

## 手順

1. **現状分析**: プロジェクトの CLAUDE.md を読み込み、現在の内容を把握
2. **問題点の特定**: 以下の観点でチェック
   - 冗長な記述がないか
   - 古くなったコードスニペットがないか
   - linter/formatterで対応すべきルールが含まれていないか
   - ファイル参照（`@path`）で置き換えられる内容がないか
3. **改善提案**: 具体的な改善点をリストアップ
4. **リファクタリング実行**: ユーザーの承認後に修正

## ベストプラクティス チェックリスト

- [ ] 簡潔さ: 150-200の指示以内に収まっているか
- [ ] ポインタ優先: コードスニペットではなく `@file` 参照を使用
- [ ] スタイルルール排除: フォーマット規則はlinter/formatterに委譲
- [ ] 構造化: 適切なセクション分けがされているか
- [ ] 重複排除: 既存ドキュメントとの重複がないか
- [ ] 最新性: 古い情報が含まれていないか

## 推奨構成

```
# プロジェクト概要
## 技術スタック
## プロジェクト構造
## 重要なファイル（@参照）
## よく使うコマンド
## コード規約
## Git ワークフロー
## テスト要件
```

## 注意事項

- 大幅な変更を行う前に必ずユーザーに確認を取る
- 変更前後の diff を提示する
- プロジェクト固有の重要な情報は保持する
