新しいカスタムスラッシュコマンドを作成してください。

## 引数

- `$1`: コマンド名（必須、ケバブケース）
- `$2`: 名前空間/サブディレクトリ（オプション）

## 作成手順

1. ユーザーにコマンドの目的・用途を質問
2. 以下の情報を確認:
   - コマンドの説明（description）
   - 引数が必要か（argument-hint）
   - 使用するツールを制限するか（allowed-tools）
3. 適切なパスにコマンドファイルを作成:
   - 名前空間あり: `~/.claude/commands/$2/$1.md`
   - 名前空間なし: `~/.claude/commands/$1.md`
4. 作成後、使用例を提示

## コマンドファイルのテンプレート

```markdown
---
description: <コマンドの説明>
argument-hint: "<引数があれば記載>"
---

<実行時の指示文>
```

## 利用可能な変数（ユーザーに説明）

| 変数 | 説明 |
|------|------|
| `$ARGUMENTS` | すべての引数 |
| `$1`, `$2`, ... | 個別の引数 |
| `!command` | シェルコマンド実行 |
| `@path` | ファイル内容埋め込み |

## 注意事項

- ファイル名はケバブケースで作成
- 既存コマンドと重複しないか確認
- descriptionは簡潔に（コマンド一覧で表示される）

コマンド名: $1
名前空間: $2
