# プルリクエスト作成

現在のブランチでプルリクエストを作成してください。

## 手順

1. 現在の状態を確認：
   - `git status` で未コミットの変更がないか確認
   - `git branch --show-current` で現在のブランチ名を取得
   - `git log origin/main..HEAD --oneline` でマージ対象のコミットを確認
   - `git diff origin/main...HEAD` で変更内容を確認

2. プロジェクトのPRテンプレートを確認：
   - `.github/PULL_REQUEST_TEMPLATE.md` が存在するか確認
   - `.github/PULL_REQUEST_TEMPLATE/` ディレクトリ内のテンプレートも確認
   - テンプレートが存在する場合、その形式に従う

3. テンプレートがない場合、既存PRのスタイルを参考にする：
   - `gh pr list --state merged --limit 3` で最近マージされたPRを確認
   - `gh pr view <number>` でPRの本文形式を確認
   - プロジェクトの慣習に合わせる

4. リモートへのプッシュ：
   - 現在のブランチがリモートに存在するか確認
   - 必要に応じて `git push -u origin <branch>` でプッシュ

5. PRの内容を分析：
   - すべてのコミットメッセージを確認（最新だけでなく全て）
   - 変更されたファイルと変更内容を把握
   - 変更の目的と影響範囲を理解

6. PRを作成：
   - `gh pr create` コマンドを使用
   - プロジェクトのテンプレート/慣習に従った形式で作成

## デフォルトのPR形式

プロジェクトにテンプレートがなく、参考にできる既存PRもない場合のみ以下を使用：

```
gh pr create --title "<タイトル>" --body "$(cat <<'EOF'
## Summary
<変更内容の要約を1-3個の箇条書きで>

## Test plan
<テスト方法のチェックリスト>
EOF
)"
```

## 注意事項

- main/masterブランチからは直接PRを作成しない
- 未コミットの変更がある場合は先にコミットを促す
- ベースブランチはデフォルトでmain（必要に応じて変更）
- 作成後、PRのURLをユーザーに報告する
- プロジェクト固有のテンプレートを最優先で尊重する
